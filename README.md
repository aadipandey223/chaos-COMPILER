# Chaos Compiler

Chaos Compiler is an educational and experimental toolkit designed to demonstrate how compilers work while simultaneously acting as a "chaos monkey" for C/C++ source code. It parses source code, visualizes the compilation pipeline, and allows for AST-level semantic/structural mutations (Chaos mode) to stress-test or obfuscate code.

## Architecture

The project is split into three main parts:
1. **Frontend (React + Vite)**: A comprehensive web interface featuring a live editor, a diff viewer for mutations, and a step-by-step **Learners Tab** that visually breaking down the compiler pipeline.
2. **Backend (Node.js + Express)**: An API server that utilizes WebAssembly (`web-tree-sitter`) to safely parse C/C++ code into an Abstract Syntax Tree (AST) without native compilation issues.
3. **Chaos Engine (C Binary)**: A backend C program that ingests the JSON AST generated by the Node server, applies configurable mutations (e.g., swapping operators, scrambling blocks), and outputs the mutated AST.

For a full system breakdown, see [architecture.md](architecture.md).

## Key Features

- **AST Generation via Tree-Sitter**: Robust parsing of complex C/C++ code into interoperable JSON ASTs.
- **Chaos Mode**: Apply targeted, AST-aware mutations to structural and logical code components.
- **Interactive Learners Tab**: Detailed, state-machine-driven visualizations of:
  1. Lexical Analysis (Tokens & DFA)
  2. Parsing (Grammar Trees)
  3. AST Building
  4. Semantic Analysis (Symbol Tables)
  5. Code Generation

## Setup & Running

**Prerequisites:** Node.js (v18+), a GCC/Clang compiler (for compiling the C mutation engine).

1. **Build the C Engine**: Navigate to `compiler/` and build the executable using `make` or the provided compile flags.
2. **Start Backend**: Navigate to `server/`, run `npm install` and `npm start` (Runs on port 4000).
3. **Start Frontend**: Navigate to `client/`, run `npm install` and `npm run dev` (Runs on Vite default port).
